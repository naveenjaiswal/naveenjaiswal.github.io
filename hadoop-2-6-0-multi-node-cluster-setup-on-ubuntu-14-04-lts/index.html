Hadoop 2.6.0 Multi Node Cluster Setup on Ubuntu 14.04 LTS

01 April 2016

Creating Hadoop cluster is not a big deal, its how precisely you follow the steps. For this, I am creating multi node cluster using Hadoop 2.6.0 (yarn) on ubuntu 14.04

Installing Java

sudo apt-get install default-jdk

check whether java is already there or not

java -version

Add a dedicated hadoop user

sudo addgroup hadoopgroup

sudo adduser --ingroup hadoopgroup hdpuser

sudo adduser hdpuser sudo

Installing SSH

sudo apt-get install ssh

Switch User

su hdpuser

sudo apt-get update

Generate keygen

ssh-keygen -t rsa -P ""

cat $HOME/.ssh/idrsa.pub >> $HOME/.ssh/authorizedkeys

Install Hadoop

sudo wget http://mirrors.sonic.net/apache/hadoop/common/hadoop-2.6.0/hadoop-2.6.0.tar.gz

sudo tar xvzf hadoop-2.6.0.tar.gz

sudo mv hadoop-2.6.0 /usr/local/hadoop

sudo chown -R hdpuser:hadoopgroup /usr/local/hadoop

Change in hosts file

find out the ip address of all master and slave nodes, by using command

ifconfig

and now make entry in host file (remove 127.0.1.1 entry from hosts file)

sudo gedit /etc/hosts

and entry accordingly

172.50.88.15 M-1923

172.50.88.7 M-1922

here M-1923 is my master node and M-1922 my slave node.

Make sure your Hostname in hosts file match with your hostname file

cat /etc/hostname

Do this on all master and slave nodes and change them accordingly

find java location

update-alternatives --config java

Edit ~/.bashrc file

sudo gedit ~/.bashrc

append the following to the end of ~/.bashrc:

export JAVA_HOME=/usr/lib/jvm/java-7-openjdk-amd64

export HADOOP_INSTALL=/usr/local/hadoop

export PATH=$PATH:$HADOOP_INSTALL/bin

export PATH=$PATH:$HADOOP_INSTALL/sbin

export HADOOPMAPREDHOME=$HADOOP_INSTALL

export HADOOPCOMMONHOME=$HADOOP_INSTALL

export HADOOPHDFSHOME=$HADOOP_INSTALL

export YARNHOME=$HADOOPINSTALL

export HADOOPCOMMONLIBNATIVEDIR=$HADOOP_INSTALL/lib/native

export HADOOPOPTS="-Djava.library.path=$HADOOPINSTALL/lib"

note that the JAVA_HOME path should be set just before '/jre'

source ~/.bashrc

Edit masters and slaves file

do this only on master node

cd /usr/local/hadoop/etc/hadoop

sudo gedit masters

here, provide the hostname of master node (in my case , it is M-1923)

sudo gedit slaves

add all slave node hostname along with master node separated by new line.
( remove master node hostname if you dont want your master node to do processing )

no need to change masters and slaves file on slave node

Edit hadoop-env.sh

sudo gedit hadoop-env.sh

replace the below line with previous #JAVA_HOME

export JAVA_HOME=/usr/lib/jvm/java-7-openjdk-amd64

do change this on both master and slave nodes

Edit core-site.xml

sudo gedit core-site.xml

create temp directory

sudo mkdir -p /home/hdpuser/tmp

sudo chown hdpuser:hadoopgroup /home/hdpuser/tmp

put below code between configuration tags 

<property>  
     <name>hadoop.tmp.dir</name>  
     <value>/home/hdpuser/tmp</value>  
     <description>Temporary Directory.</description>
</property>

<property>  
     <name>fs.defaultFS</name>  
     <value>hdfs://M-1923:54310</value>  
</property>

here M-1923 is master node's hostname
do change this on both master and slave nodes

Edit mapred-site.xml

only on master node

sudo gedit mapred-site.xml

put below code between configuration tags 

<property>  
     <name>mapreduce.jobtracker.address</name>
     <value>M-1923:54311</value>
</property>


<property>  
     <name>mapreduce.framework.name</name>
     <value>yarn</value>
     <description>The framework for running mapreduce jobs</description>
</property>

here M-1923 is master node's hostname
do this change only on master node

Edit yarn-site.xml

sudo gedit yarn-site.xml

put below code between configuration tags 

<property>  
     <name>yarn.nodemanager.aux-services</name>
     <value>mapreduce_shuffle</value>
</property>

<property>  
     <name>yarn.resourcemanager.scheduler.address</name>
     <value>M-1923:8030</value>
</property> 

<property>  
     <name>yarn.resourcemanager.address</name>
     <value>M-1923:8032</value>
</property>

<property>  
      <name>yarn.resourcemanager.webapp.address</name>
      <value>M-1923:8088</value>
</property>

<property>  
      <name>yarn.resourcemanager.resource-tracker.address</name>
      <value>M-1923:8031</value>
</property>

<property>  
      <name>yarn.resourcemanager.admin.address</name>
      <value>M-1923:8033</value>
</property>

here M-1923 is master node's hostname
do this change on both master as well slave nodes

Edit hfds-site.xml

Master node configuration

sudo mkdir -p /usr/local/hadoop/hadoop_store/hdfs/namenode

sudo mkdir -p /usr/local/hadoop/hadoop_store/hdfs/datanode

sudo chown -R hdpuser:hadoopgroup /usr/local/hadoop

put below code between configuration tags 

<property>  
     <name>dfs.replication</name>
     <value>2</value>
</property>


<property>  
     <name>dfs.namenode.name.dir</name>
     <value>/usr/local/hadoop/hadoop_data/namenode</value>
</property>


<property>  
     <name>dfs.datanode.data.dir</name>
     <value>/usr/local/hadoop/hadoop_data/datanode</value>
</property>

master node configuration

Slave node configuration

sudo mkdir -p /usr/local/hadoop/hadoop_store/hdfs/datanode

sudo chown -R hdpuser:hadoopgroup /usr/local/hadoop

put below code between configuration tags 

<property>  
     <name>dfs.replication</name>
     <value>2</value>
</property>


<property>  
     <name>dfs.namenode.name.dir</name>
     <value>/usr/local/hadoop/hadoop_data/namenode</value>
</property>


<property>  
     <name>dfs.datanode.data.dir</name>
     <value>/usr/local/hadoop/hadoop_data/datanode</value>
</property>

slave node configuration

send keys to slave nodes

sudo ssh-copy-id -i ~/.ssh/id_rsa.pub hdpuser@M-1922

Format the New Hadoop Filesystem

hadoop namenode -format

this will clear all data from data nodes

Starting Hadoop

cd /usr/local/hadoop/sbin

start-dfs.sh

start-yarn.sh

jps

check what's really up and running:

on master node

10924 SecondaryNameNode  
11547 Jps  
10733 DataNode  
10579 NameNode  
11097 ResourceManager  
11238 NodeManager

on slave node

11247 Jps  
10463 DataNode  
15638 NodeManager

your hadoop cluster now up nad running

http://m-1923:8088/

http://m-1923:50070/

test the wordcount or pi program

if you found some error or typo , please do comment it out
